## 零、真智能体架构说明

### 核心模式

```
不是：Agent.process() → 写死的 switch/case → 调方法 → 返回
而是：
  AgentEngine.run():
    messages = [systemPrompt, userInput]
    while True:
      response = LLM.reason(messages, tools=agent.getTools())
      if response.isFinalAnswer → return
      for toolCall in response.toolCalls:
        result = ToolRegistry.execute(toolCall)
        messages += result
```

### 新增文件一览

| 文件 | 包 | 行数 | 功能 |
|------|---|:----:|------|
| `ToolSchema.java` | `agent` | ~60 | 工具描述（转 OpenAI function calling 格式） |
| `ToolRegistry.java` | `agent` | ~100 | 工具注册中心，所有 Agent 注册其 Tool |
| `AgentEngine.java` | `agent` | ~170 | LLM 推理循环（while + tool calling） |
| `Agent.java` | `agent` | 改接口 | 加 getSystemPrompt() / getTools() / registerTools() |

### 每个 Agent 的三件套

每个 Agent 现在多出 3 个接口方法：

```java
// 例：ResourceFacadeAgent
String getSystemPrompt(task)     → "你是教学资源生成专家..."
List<ToolSchema> getTools()       → [generate_lesson, generate_quiz, ...]
void registerTools(registry)      → 把每个工具的实际执行方法注册到 ToolRegistry
```

原有 `process()` 方法保留不动，向后兼容。

### ToolRegistry 注册表

```
ToolRegistry
  ├─ orchestrator      → 6 tools (route_intent / greet / handle_help /
  │                                  handle_learning_path / handle_resource_generate /
  │                                  handle_media_generate)
  ├─ resourcefacade    → 5 tools (generate_lesson / quiz / code_case /
  │                                  mind_map / extended_reading)
  ├─ learnerprofile    → 2 tools (get_profile / extract_profile)
  ├─ learningpath      → 1 tool  (generate_path)
  ├─ qualityassessment → 1 tool  (assess_full 含4层检查)
  ├─ effectassessment  → 2 tools (evaluate_quiz / analyze_weakness)
  ├─ qa_agent          → 2 tools (answer / generate_mermaid)
  └─ contentanalyzer   → 1 tool  (analyze)
  └─ (总计: 20 tools, 8 agents)
```

AgentEngine 在推理循环中通过 ToolRegistry 查找并执行工具。

## 二、智能体详细设计

### 2.0 核心引擎 — AgentEngine

```java
@Component
public class AgentEngine {
    public AgentResult run(Agent agent, AgentTask task, String... extraContext) {
        // 1. system prompt + user input → messages
        // 2. agent.getTools() → tools (OpenAI function calling 格式)
        // 3. loop (max 10 rounds):
        //      LLM.reason(messages, tools)
        //      if tool_calls → ToolRegistry.execute() → messages += result
        //      if final → return
    }
}
```

| 组件 | 状态 | 说明 |
|------|:----:|------|
| `AgentEngine.java` | ✅ | 推理循环核心，编译通过 |
| Tool Schema 序列化 | ✅ | `ToolSchema.toOpenAISpec()` 转 OpenAI 格式 |
| 推理循环实现 | ✅ | while(true) + tool_calls 解析 + 上下文追加 |

**当前 Agent 调度方式**（两套并存）：

```
AgentBus.dispatch()  ← 当前在用（不改现有流程）
  └─ 直接调 agent.process(task)

AgentEngine.run()    ← 新引擎（需手动切换）
  └─ LLM 推理循环 + tool calling + 工具执行
```

AgentEngine 是一个独立的 `@Component`，**没有接入现有调度流程**。改 `AgentBus.java` 一行代码即可切换。
