## 十二、按项目阶段（P0～P4）

### P0 — 聊天打通（最高优先级）

> **阶段目标**：学生工作台不再「连接中」，能流式对话并在 Timeline 看到智能体步骤。

| 任务 | 说明（一句话） | 状态 |
|------|----------------|------|
| `chat` 包：`ChatWebSocketHandler` | 实现 `/ws/chat` 端点：接收前端消息、校验 JWT、把回复推回浏览器 | ❌ |
| `AiServerWsProxy` | 后端作为中间层连接 ai-server 的 WebSocket，把它的流式输出转成我们的 `chunk`/`done` 协议 | ❌ |
| `ChatSessionService` | 管理 `chat_session` 表：创建会话 ID、关联用户、记录 ai-server 侧 session | ❌ |
| `AgentRunLogService` | 每次智能体调用写入 `agent_run_log`，并向前端推 `agent_step` 事件 | ❌ |
| 默认 `qa_agent` 流式答疑 | P0 先不做复杂路由：用户发消息 → 直接走答疑通道（转发 ai-server 对话） | ❌ |
| **阶段验收** | `student1` 在工作台发消息能逐字显示回复；Timeline 至少有 1～2 步；库里有 `agent_run_log` | ❌ |

### P1 — 路径与资源（MVP 1～4 步）

> **阶段目标**：说出学习目标后出现**真实**学习路径，点节点能生成讲义/题目。

| 任务 | 说明（一句话） | 状态 |
|------|----------------|------|
| `orchestrator` 规则路由 | 协调智能体：根据关键词判断走「路径 / 出题 / 资源 / 默认答疑」，再调用对应模块 | ❌ |
| `KnowledgeGraphService` + 数据入库 | 把 `knowledge_graph.json` 载入 `knowledge_point`/`knowledge_edge`，提供前置依赖与路径搜索 | ❌ |
| `LearningPathService` 真逻辑 + 落库 | 按图谱算节点顺序 + 可选 LLM 写计划说明，结果写入 `learning_path` 与 `step` 表 | ❌ |
| `GET /api/learning-path/current` | 返回该学生当前进行中路径（刷新页面后右侧栏仍能显示） | ❌ |
| `resourcefacade` + `POST /api/resources/generate` | 点击节点生成讲义/题目/代码；`ILLUSTRATION` 走生图 | ❌ |
| `mediageneration` + 媒体存储 | 调生图 API，写 `media_url`；文件落 `storage/media/` 或对象存储 | ❌ |
| WS 推送 `artifact`（路径/资源/媒体） | 下发路径 JSON、资源卡片、`media_image` 等 | ❌ |
| **阶段验收** | 点节点能出讲义或题目；**至少一种知识点能出示意图** | ❌ |

### P2 — 答题与画像（MVP 5～6 步）

> **阶段目标**：做完题能更新掌握情况，左侧画像有变化，路径推荐可跟着调。

| 任务 | 说明（一句话） | 状态 |
|------|----------------|------|
| `learnerprofile` 模块 | 读写 `learner_profile`：掌握度、目标、常见错误等；各智能体按规则更新字段 | ❌ |
| `POST /api/quiz/submit` | 提交答案 → 判分/记录 `quiz_record` → 触发画像与效果评估 | ❌ |
| `GET /api/profile/summary` | 工作台左侧「我的画像」卡片的数据源 | ❌ |
| `qualityassessment` | 文本资源检索/LLM 校验；**媒体**校验尺寸/时长/敏感词 | ❌ |
| WS `profile_patch` | 答题或聊天后通过 WS 推送画像增量，前端无需整页刷新 | ❌ |
| `chat_message` MULTI_CARD | 答疑回复可含图片卡片 JSON（`content_type=MULTI_CARD`） | ❌ |
| **阶段验收** | 提交一道题后画像数字或文案变化；路径下一步推荐有依据 | ❌ |

### P3 — 上传进库 + 管理监控（MVP 7 步）

> **阶段目标**：上传的资料真正进入知识库；管理员能看到智能体运行情况。

| 任务 | 说明（一句话） | 状态 |
|------|----------------|------|
| `uploadDocument` 实现 | 补全 `AiServerClient.uploadDocument`：把队列里的文件发给 ai-server 做 RAG 入库 | ❌ |
| `contentanalyzer` + 写分析结果 | 上传处理完成后解析文档、映射知识点，写 `analysis_result` 与 `mapped_kp_ids` | ❌ |
| `KbUploadTask` 实体增加 v2 字段 | JPA 实体对齐 SQL 中的 `analysis_result`、`mapped_kp_ids`，Admin/学生能读到 | ❌ |
| `effectassessment` + 行为/路径调整日志 | 记录停留、错题等行为到 `study_behavior`，按规则改路径并写 `path_adjustment_log` | ❌ |
| Admin `agent-stats` / `agent-runs` | 管理后台 API：统计各智能体调用次数/失败率、分页查最近运行日志 | ❌ |
| **阶段验收** | 上传 PDF 后队列 COMPLETED 且库里有映射知识点；Admin 概览能看到 Agent 统计 | ❌ |

### P4 — 加分项（含生视频）

> **阶段目标**：体验与可维护性提升；**生视频**、完整会话回放放本阶段。

| 任务 | 说明（一句话） | 状态 |
|------|----------------|------|
| `VIDEO_CLIP` 生视频 | `mediageneration` 接生视频 API，写 `resource_item`，WS 推 `media_video` | ❌ |
| 管理端媒体 API 配置 | `media.image.*` / `media.video.*` 写入 `sys_config`，Admin 可配 | ❌ |
| LLM 意图分类（替代纯规则） | 用模型判断「要路径 / 要题 / 要配图 / 要视频 / 闲聊」 | ❌ |
| 路径动态调整规则 | 根据 P2 行为数据自动插复习节点、补代码案例等 | ❌ |
| `chat_message` 全量持久化 | 含 `MULTI_CARD` 多模态消息的历史回放 | ❌ |
