## 十三、智能体 vs ai-server 依赖矩阵

说明每个智能体的实现来源，方便后续开发决策。

### 图例

| 标记 | 含义 |
|------|------|
| 🏗️ 从头搭建 | 纯本地实现，不依赖 ai-server。改业务逻辑、数据模型 |
| 🔗 调 ai-server API | 通过 `AiServerClient`（REST）或 `AiServerWsProxy`（WS）调 DeepTutor。需要 ai-server 在线 |
| 🧩 组合实现 | 调多个本地服务 + 可选 ai-server。涉及跨模块调用 |
| 📡 仅心跳 | 只调 `ping()` 做健康探测，无 AI 能力。计划中后续升级 |

### 智能体依赖矩阵（架构规范 vs 当前实现 vs 推荐方案）

| 智能体 | 架构规范要求 | 当前实现 | 差距 | 推荐方案 |
|--------|------------|---------|------|---------|
| **OrchestratorAgent**（总调度） | 任务分发+结果聚合+上下文管理；禁止交叉调用，星型拓扑 | 仅关键词意图识别。**路由/聚合在 ChatWebSocketHandler** | 🚫 待 P4 改造 | **🏗️ 从头搭建**。不做 AI。P4 移至 OrchestratorAgent 统一调度 |
| **QaAgent**（答疑） | RAG 问答 + 图文回答 + **Mermaid 流程图** | `AiServerWsProxy` WS 流式文本转发。RAG 由 DeepTutor 内部做 | 🟡 文本已通，缺 Mermaid | **🔗 调 ai-server**（已有）。P6 加多模态 |
| **LearningPathAgent**（路径规划） | LLM 动态规划 ≥5 步骤 + **关联资源类型** | BFS 图谱遍历，固定路径。未关联资源类型 | 🚫 LLM+资源关联 均未实现 | **🏗️ 保持 BFS**。教育路径需要确定性，不追 LLM。P6 加资源关联 |
| **ResourceFacadeAgent**（资源生成） | **LLM 并行生成** 5 种资源（文档/题目/代码/思维导图/图文脚本） | 模板内容，单线程 | 🚫 差距大 | **📡→🔗 P5 重点升级**。调 DeepTutor book engine 生成真实内容 |
| **LearnerProfileAgent**（画像） | LLM Prompt 从对话抽取 6 维度 + **PostgreSQL+Chroma 混合存储** | 答题记录自动计算。无 Chroma | 🚫 LLM+向量库 均未实现 | **🧩 混合**。答题记录为主（准）。P5 可选 LLM 补充。Chroma 最后加 |
| **QualityAssessmentAgent**（质检） | RAG 比对 + LLM 自检 + **讯飞安全 API** + **学术规范性检查（拦截虚假引用）** | 非空/格式检查 | 🚫 差距大 | **🏗️ 保持本地**。RAG 质检过度设计。P6 加讯飞安全 API |
| **ContentAnalyzerAgent**（内容分析） | LLM 提取知识点 | 关键词匹配 | 🚫 有差距 | **🏗️ 保持关键词**。课程结构化，关键词足够。P5 可选 LLM 补充 |
| **EffectAssessmentAgent**（效果评估） | LLM 分析薄弱点 + **追踪资源点击+停留时间** → 路径调整 | 仅低分→插复习节点。无行为追踪 | 🚫 行为追踪未实现 | **🧩 保持本地**。当前规则够用。P5 加行为追踪表 |
| **MediaGenerationAgent**（媒体生成） | 生图 + **生视频** + **TTS**（语音合成） | 生图 mock。无视频，无 TTS | 🚫 差距大 | **🏗️→🔗 P6 升级**。对接第三方生图/生视频 API。TTS 可选 |

### 核心结论

1. **架构规范全都要 LLM，实际只有 QaAgent 真调了 AI。** 这是最大差距。

2. **哪些该追，哪些不该追（按优先级排序）：**

| 优先级 | 方向 | 目标智能体 | 原因 |
|--------|------|-----------|------|
| 🥇 P5 必做 | 资源生成接 LLM | ResourceFacadeAgent | 模板内容最影响体验。调 DeepTutor book engine 生成真实资源 |
| 🥇 P6 必做 | 生图/生视频 API | MediaGenerationAgent | 媒体是规范硬要求，当前只有 mock |
| 🥈 P6 必做 | 讯飞安全 API | QualityAssessmentAgent | 内容安全是上线底线。RAG 校验现阶段过度设计 |
| 🥉 P5 可选 | LLM 补充对话画像 | LearnerProfileAgent | 答题记录够准。LLM 抽取容易抽噪声。Chroma 向量库不追 |
| 🥉 P5 可选 | LLM 补充知识点提取 | ContentAnalyzerAgent | 结构化课程关键词匹配已够。LLM 收益有限 |
| 🥉 P5 可选 | 行为追踪表 | EffectAssessmentAgent | 低分→复习节点简单有效。行为表只加存储字段 |
| ❌ 不追 | LLM 动态路径规划 | LearningPathAgent | 教育路径需要确定性。BFS 比 LLM 可靠。P6 加资源关联即可 |
| ❌ 不追 | RAG 事实性校验 | QualityAssessmentAgent | 过度设计。格式检查 + 讯飞 API 够了 |
| ❌ 不追 | Chroma 向量库 | LearnerProfileAgent | PostgreSQL 够用。向量库对画像抽取收益不大 |

3. **OrchestratorAgent** = 永远不调 AI，职责就是路由决策 + 重试 + 质量闸门。

4. 后续开发新 agent：先看架构规范想要什么，再看这个项目需不需要。**不是规范写的都要做。**
