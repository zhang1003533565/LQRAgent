# 后端打通指南

> 本文档描述从当前状态到完整多智能体系统的每一步操作。
> 每一步只写：做什么、改哪里、验证什么。

---

## 第一步：AgentBus 切换到 AgentEngine

**目标**：让所有 Agent 走 LLM 推理循环，而不是直接调 `process()`

**改什么**：
- `AgentBus.java` 的 `dispatch()` 方法
- 当前：`agent.process(task)` → 直接执行
- 改为：`agentEngineFactory.getObject().run(agent, task, sessionId)` → 走推理循环

**验证**：
- 启动后端，调用任意 Agent 的 API
- 日志中应出现 `[AgentEngine] round=0, agent=xxx` 的推理循环日志
- 如果 LLM 未配置，应回退到 `process()` 降级

---

## 第二步：Orchestrator 完全走 LLM 决策

**目标**：Orchestrator 不再用 switch/case 路由，完全由 LLM 自主决策调用哪些工具

**改什么**：
- `OrchestratorAgent.java` 的 `process()` 方法
- 当前有两套逻辑：`process()` 里的 switch/case + `registerTools()` 里的工具注册
- 切换到 AgentEngine 后，`process()` 只作为 LLM 未配置时的降级入口
- `getSystemPrompt()` 已经写好，LLM 会根据 prompt 自主选择工具

**验证**：
- 发送"我想学Python装饰器"→ 日志应显示 LLM 选择 `handle_learning_path` 工具
- 发送"生成讲义"→ 日志应显示 LLM 选择 `handle_resource_generate` 工具
- 发送"你好"→ 日志应显示 LLM 选择 `greet` 工具

---

## 第三步：子Agent派发链路打通

**目标**：Orchestrator 调用子 Agent 时，子 Agent 也走 AgentEngine 推理循环

**改什么**：
- `OrchestratorAgent.registerTools()` 中的 `dispatchToAgent()` 方法
- 当前已经是 `agentBus().dispatch(subTask).join()` 同步调用
- 切换 AgentBus 后，子 Agent 会自动走 AgentEngine

**验证**：
- Orchestrator 收到"我想学装饰器"→ 调 `handle_learning_path` → AgentBus 派发给 LearningPathAgent → LearningPathAgent 走 AgentEngine 推理循环 → 调 `generate_path` 工具 → 返回路径
- 日志应显示完整的调用链：Orchestrator → LearningPathAgent → generate_path

---

## 第四步：路径规划 + LLM 个性化排序

**目标**：BFS 生成路径后，用 LLM 根据学生画像做个性化排序

**改什么**：
- `LearningPathService.java` 的 `generatePath()` 方法
- 当前：BFS 找最短路径 → 直接返回
- 改为：BFS 找路径 → 调 LLM 排序（根据学生画像调整顺序）→ 返回

**验证**：
- 生成路径后，日志应显示 LLM 排序调用
- 不同画像的学生，同一目标的路径顺序应不同

---

## 第五步：效果评估 → 路径自动更新

**目标**：学生答题后，评估薄弱点并自动在路径中插入复习节点

**改什么**：
- `EffectAssessmentService.java` 的 `evaluateQuizResult()` 方法
- 当前：记录答题数据 → 分析薄弱点
- 改为：记录答题 → 分析薄弱点 → 调 `LearningPathService` 在路径中插入复习节点

**验证**：
- 学生答错某知识点 → 路径中自动出现该知识点的复习步骤
- 路径 API 返回的步骤列表中包含新插入的复习节点

---

## 第六步：答疑 Agent 调用 Python ai-server

**目标**：答疑时调用 Python 侧的 RAG、对话能力

**改什么**：
- `QaAgentService.java` 的 `handleMessage()` 方法
- 当前通过 `AiServerWsProxy` WebSocket 连接 Python
- 验证 WebSocket 连接是否正常工作

**如果 WebSocket 不通**：
- 改用 HTTP 调用：`POST http://localhost:8001/api/v1/plugins/tools/rag/execute`
- 创建 `AiServerHttpClient.java` 封装 HTTP 调用

**验证**：
- 学生提问"什么是装饰器" → 能收到文字回答
- 回答内容应包含相关知识点的检索结果

---

## 第七步：质量评估 → 重试机制

**目标**：资源生成后自动质检，不合格时触发重试

**改什么**：
- `ResourceGenerationService.java` 的 `generate()` 方法
- 当前：生成 → 保存 → 返回
- 改为：生成 → 保存 → 调 `QualityAssessmentService.assessFull()` → 不合格则删除旧内容重新生成一次

**验证**：
- 正常资源生成后，日志应显示质检通过
- 故意构造不合格内容（如空内容），应触发重试

---

## 第八步：敏感词库 + 学术规范库填充

**目标**：让质量评估的敏感词和学术规范检查真正生效

**改什么**：
- `SensitiveFilter.java` — 填充敏感词列表（中文敏感词 + 课程相关违规词）
- `AcademicChecker.java` — 填充学术规范规则（禁止虚假引用、绝对化用语等）

**验证**：
- 生成包含敏感词的资源 → 质检应拦截
- 生成包含"绝对""100%保证"等绝对化用语的资源 → 质检应拦截

---

## 第九步：数据库初始化 + 知识图谱导入

**目标**：确保数据库表结构正确，知识图谱数据导入成功

**做什么**：
1. 启动 Spring Boot，JPA 自动建表
2. `KnowledgeGraphLoader` 启动时读取 `knowledge_graph.json` 并导入
3. 验证 `knowledge_point` 表有 30+ 条记录
4. 验证 `knowledge_edge` 表有依赖关系

**验证**：
- 调用 `GET /api/learning-path` 相关接口，返回的数据应引用知识点 ID
- 调用知识图谱查询接口，应返回正确的前置/后置关系

---

## 第十步：Python ai-server 集成（增强能力）

**目标**：Java 侧调用 Python 的 RAG、Web Search 等成熟工具

**做什么**：
1. 创建 `AiServerClient.java`，封装 HTTP 调用
2. 在 `IntelligentQaAgent` 中使用（替代或补充 WebSocket）
3. 在 `ResourceGenerationAgent` 中可选调用 Python RAG 做知识检索

**HTTP 接口**：
- `GET http://localhost:8001/api/v1/plugins/list` — 获取可用工具列表
- `POST http://localhost:8001/api/v1/plugins/tools/rag/execute` — 调用 RAG
- `POST http://localhost:8001/api/v1/plugins/tools/web_search/execute` — 调用搜索

**验证**：
- Java 调用 Python RAG → 返回知识库检索结果
- 答疑 Agent 回答问题时，内容应包含 RAG 检索的参考资料

---

## 第十一步：媒体生成（预留 Key 接入位）

**目标**：图片生成支持 DALL·E 3 / Stability AI，视频生成预留接口

**改什么**：
- `MediaGenerationService.java` 已支持 `dalle3` 和 `sd3` 模式
- 配置 API Key：在 `sys_config` 表或环境变量中设置 `agent.mediagen.image_provider` 和 `agent.mediagen.api_key`
- 视频生成：在 `MediaGenerationService` 中添加 `video` 类型，预留 SeeDance / TTS 接口

**验证**：
- 配置 DALL·E 3 Key 后，调用 `POST /api/media/generate` → 返回真实图片 URL
- 未配置时返回 mock 占位符

---

## 第十二步：端到端场景验证

**目标**：完整走通所有核心场景

**场景1：新学生完整流程**
1. 学生注册/登录
2. 发送"我想学Python装饰器" → Orchestrator 识别意图 → 调 LearningPathAgent → 返回学习路径
3. 点击路径中的知识点 → 调 ResourceGenerationAgent → 生成讲义/题目/代码
4. 资源经过质量评估 → 通过后返回给学生

**场景2：效果评估联动**
1. 学生做练习题 → 答错
2. 调 EffectAssessmentAgent → 评估薄弱点
3. 路径自动更新 → 插入复习节点

**场景3：智能答疑**
1. 学生提问"装饰器和注解有什么区别"
2. 调 IntelligentQaAgent → 检索知识库 → 生成回答 + Mermaid 流程图

**场景4：防幻觉**
1. 故意让 LLM 生成错误内容（如"装饰器只在Python 2中可用"）
2. QualityAssessmentAgent 的 LLM 事实性校验应拦截
3. 触发重新生成
