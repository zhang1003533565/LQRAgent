# 地基阶段任务清单

## 目标

这一阶段不追求功能完整，只追求把后续所有开发都需要的基础设施和基础数据定下来。

## 第一批必须完成的事项

### 1. 课程数据地基

- 确定 Python 课程 6 个以上章节
- 为每个知识点分配唯一 `kp_id`
- 产出 `knowledge_graph.json`
- 准备公共知识库种子文件

负责人建议：

- 课程数据负责人

## 2. 角色与登录地基

- 明确三个角色：`student`、`teacher`、`admin`
- 登录后返回角色信息
- 前端根据角色自动跳转

负责人建议：

- 后端接口负责人
- 前端负责人

## 3. React 前端地基

- 初始化 React 项目结构
- 建立基础路由
- 建立全局鉴权状态
- 建立 WebSocket 基础封装

负责人建议：

- 前端负责人

## 4. backend 业务地基

- 建立 Spring Boot 基础目录结构
- 建立 `AiServerClient`
- 建立 `LearningController`
- 建立 `UploadQueueController`
- 建立基础 DTO

负责人建议：

- 后端负责人

## 5. ai-server 接入地基

- 跑通 `knowledge` 路由
- 跑通 `unified_ws` 流式聊天
- 跑通 `question` 题目生成
- 跑通 `solve` 课程答疑

负责人建议：

- AI 接入负责人

## 6. 上传队列地基

- 建 `kb_upload_task` 表
- 做一个单机 worker
- 实现 `PENDING / PROCESSING / COMPLETED / FAILED`
- 前端能看到任务状态

负责人建议：

- 后端负责人

## 7. 学习路径地基

- 定义路径数据结构
- 实现图谱查询
- 实现最小路径生成逻辑

负责人建议：

- 路径规划负责人

## 建议的开工顺序

1. 课程数据与图谱
2. 登录与角色
3. React 基础页面壳子
4. backend 统一接口壳子
5. ai-server 四个关键能力接通
6. 上传队列
7. 最小学习路径

## 地基阶段验收标准

- 可以登录并识别角色
- 可以进入学生工作台
- 可以发起一次学习请求
- 可以返回一条学习路径
- 可以上传个人资料并看到排队状态
- 可以调用 ai-server 返回一段流式回答
