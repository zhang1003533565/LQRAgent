## 第 1 页｜封面

**页面内容**
- 主标题：**LQRAgent — 面向 Python 程序设计的个性化 AI 学习系统**
- 副标题：第十五届软件杯 A3 赛题 / Edu.AI 学生端
- 团队信息、学校、日期

**讲解要点（可不在 PPT 上写满）**
- 一句话定位：不是「聊天机器人」，而是 **多 Agent 驱动的个性化学习平台**

**视觉建议**
- 浅蓝教育风 + 产品 Logo（Edu.AI）
- 背景可用：聊天界面 + 知识图谱缩略图拼贴

---

## 第 2 页｜背景与痛点

**页面内容**
- **赛题要求**：Python 高级语言程序设计 · 个性化学习 · 智能答疑 · 路径规划
- **三类痛点**
  1. 传统平台：内容固定，难以因材施教
  2. 通用 ChatGPT：无课程知识库，易幻觉、无学习闭环
  3. 学习数据分散：路径、练习、画像、资料各自孤立

**讲解要点**
- 我们的目标：**「问—学—练—评—改」闭环**，且每个环节有 AI 参与但可控

**视觉建议**
- 左文字右对比图：「通用 AI」vs「LQRAgent」

---

## 第 3 页｜系统总体架构

**页面内容**
- 三层架构图（自上而下）：
  1. **学生端 React**：聊天 / 路径 / 资源 / 答题 / 图谱 / 上传 / 画像
  2. **Java 后端 Orchestrator**：PlanningAgent · Pipeline · 20+ Agent · WebSocket 流式
  3. **AI Server + RAG**：向量检索 · 讲义/题目生成 · 知识库

**讲解要点**
- 强调：**意图识别 → Pipeline 编排 → 多 Agent 分工**，不是单模型一把梭

**视觉建议**
- 用箭头标注数据流：`用户提问 → WS → 规划 → Agent 链 → 流式返回 + Artifact 卡片`

---

## 第 4 页｜多 Agent 协作机制

**页面内容**
- **调度层**：PlanningAgent（Function Calling 意图识别）
- **学习层**：Profile / Path / Quiz / QA / Resource / Quality …
- **Phase 2 协商**：路径生成、出题设计多轮协商
- **Phase 3 督导**：Supervisor 审核路径与题目质量
- **防幻觉**：RAG 检索优先 · 引用来源展示 · 质量 Agent 后置检查

**讲解要点**
- 用一张「Agent 调用链」截图（聊天里折叠前的步骤条）作为右侧配图

**视觉建议**
- 表格或同心圆：用户层 / 内容层 / 服务层（精简为 8～10 个 Agent）

## 第 13 页｜创新点与总结

**页面内容**
- **创新点（4 条）**
  1. 多 Agent Pipeline + 协商/督导，非单轮 Prompt
  2. 学习全链路打通：聊 → 路径 → 资源 → 练 → 图谱 → 画像
  3. RAG 私有知识库 + 引用可验证，降低幻觉
  4. 流式体验 + Agent 步骤透明可折叠，结果可解释
- **技术栈**：React · Spring Boot · WebSocket · MySQL · Redis · DeepSeek · ai-server
- 结束语：**让 AI 成为「可信赖的学习伙伴」，而不是「会聊天的搜索引擎」**

**视觉建议**
- 简洁大字 + 二维码（可选：演示环境 / 仓库地址）
