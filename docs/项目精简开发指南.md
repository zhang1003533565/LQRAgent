# LQRAgent 项目精简开发指南

## 一、目标是什么

做一个**面向软件杯比赛的 AI 课程学习系统**，课程固定为《Python 高级语言程序设计》。

核心卖点四个：

- 多智能体协同
- 个性化学习路径
- 资源自动生成
- 基于行为反馈的动态调整

**最小闭环（MVP）**：

1. 学生登录，进入聊天式工作台
2. 输入学习目标（如"学习 Python 装饰器"）
3. 系统根据公共知识图谱生成学习路径
4. 点击路径节点，系统生成讲解文档、题目、代码案例
5. 学生提交答题结果
6. 系统更新画像，调整下一步推荐
7. 学生继续追问，系统结合公共知识库和个人知识库回答

只要这 7 步稳定跑通，软件杯演示就成立了。

## 二、三层架构分工

```
前端 (React)  →  后端 (Spring Boot)  →  AI服务 (Python/FastAPI)
```

| 层级 | 技术栈 | 核心职责 |
|------|--------|----------|
| 前端 | React | 登录、聊天工作台、路径展示、资源卡片、答题、上传队列 |
| 后端 | Spring Boot 3 / Java 21 | 用户鉴权、知识图谱、学习路径、画像、上传队列、封装 AI 调用 |
| AI服务 | Python / FastAPI（已有） | 大模型调用、流式对话、RAG 检索、资源生成、题目生成、答疑、记忆 |

**调用链**：前端请求 → backend 查角色/画像/图谱 → 按意图路由到 AI 服务 → 收敛结果更新画像 → 返回前端。

> AI 服务已有现成能力，**不要重写**，直接复用。

## 三、后端怎么做

### 3.1 技术栈

- Spring Boot 3.2.5 + Java 21
- 数据库：MySQL（建议）
- 启动时自动拉起 AI 服务进程（`AIServerRunner` 已实现）

### 3.2 需要建的模块

| 模块 | 对应智能体 | 职责 |
|------|-----------|------|
| `auth` / `user` | — | 登录鉴权、角色识别（student / teacher / admin），登录后返回角色和跳转路由 |
| `orchestrator` | 协调智能体 | 接收前端请求，判断意图，路由到对应 AI 能力，收敛结果返回前端 |
| `content-analyzer` | 内容分析智能体 | 对上传的课程资料调用 `rag` 工具做内容解析，抽取概念、关键词、知识点映射关系，结果写回知识图谱 |
| `knowledge-graph` | — | 维护课程章节、知识点、前置依赖关系，提供路径搜索函数 |
| `learner-profile` | 学生画像智能体 | 结构化画像存储（掌握程度、学习目标、认知风格、常见错误等），结合 ai-server memory 从答题和聊天中持续更新 |
| `learning-path` | 路径规划智能体 | 图谱算候选路径 + 画像决定节奏 + 大模型生成自然语言计划，支持动态调整 |
| `upload-queue` | — | 文件上传异步排队，状态流转 PENDING → PROCESSING → COMPLETED / FAILED |
| `ai-server-client` | — | 封装对 AI 服务的所有 HTTP/WebSocket 调用，隔离前端与 AI 服务的耦合 |
| `resource-facade` | 资源生成智能体 | 聚合资源生成结果（讲义、思维导图、题目、代码案例），调用 `book` / `question` / `deep_question` |
| `quality-assessment` | 质量评估智能体 | 对生成内容做三段式校验：事实检索校验 → 规则校验（敏感词过滤）→ LLM 自检（validation 阶段 + `code_execution` 验证代码正确性），不通过则带原因重试一次 |
| `effect-assessment` | 效果评估智能体（加分项） | 收集答题正确率、停留时间、重复追问点等行为指标，生成评估报告，触发路径动态调整规则 |
| `qa-agent` | 答疑智能体（加分项） | 调用 `deep_solve` 管道 + `code_execution`，支持 Python 装饰器、Java 注解等复杂问题的步骤化解答 |

### 3.3 知识图谱三个核心函数

```java
getPrerequisites(kpId)              // 获取前置知识点
findLearningPath(current, target)   // 找从当前到目标的路径
recommendNextPoints(profile, state) // 根据画像推荐下一步
```

### 3.4 上传队列表结构

表名：`kb_upload_task`

| 字段 | 说明 |
|------|------|
| `id` | 主键 |
| `user_id` | 上传用户 |
| `file_name` | 文件名 |
| `kb_scope` | PUBLIC / PERSONAL |
| `status` | PENDING / PROCESSING / COMPLETED / FAILED |
| `priority` | 优先级 |
| `error_message` | 失败原因 |
| `created_at` / `started_at` / `finished_at` | 时间戳 |

第一版用数据库任务表 + Spring Boot 定时轮询 worker 即可，不需要上 Redis/RabbitMQ。

### 3.5 其他核心数据表

`user`、`role`、`learner_profile`、`learning_goal`、`knowledge_point`、`knowledge_edge`、`learning_path`、`learning_path_step`、`resource_item`、`quiz_record`、`study_behavior`、`path_adjustment_log`

### 3.6 课程数据文件（需手动准备）

放在 `backend/src/main/resources/course/`：

```
outline.json          # 课程章节结构
knowledge_graph.json  # 知识点 + 依赖关系（30+ 知识点）
seed_kb/              # 公共知识库种子文件（讲义、例题、代码）
examples/             # 代码示例
```

## 五、八个智能体与 AI 服务的对应关系

| 智能体 | 实现位置 | 复用的 AI 服务能力 | 备注 |
|--------|---------|-------------------|------|
| 协调智能体 | backend `orchestrator` | `unified_ws.py` / Session Runtime | 统一入口，按意图分发 |
| 内容分析智能体 | backend `content-analyzer` | `rag` 工具 + `knowledge.py` | 解析上传资料，抽取概念写回图谱 |
| 路径规划智能体 | backend `learning-path` | `book.py` (Book Engine) | 图谱算路径，大模型生成自然语言计划 |
| 资源生成智能体 | backend `resource-facade` | `book.py` / `question.py` / `deep_question` | 生成讲义、题目、代码案例、闪卡 |
| 质量评估智能体 | backend `quality-assessment` | `deep_question` validation 阶段 + `code_execution` | 三段式校验，失败重试一次 |
| 学生画像智能体 | backend `learner-profile` | `memory.py` / `sessions.py` | 结构化画像 + 文本记忆双轨 |
| 答疑智能体（加分项） | backend `qa-agent` | `deep_solve` + `code_execution` | 步骤化解答，支持代码运行验证 |
| 效果评估智能体（加分项） | backend `effect-assessment` | Question Bank 错题回顾 + `memory.py` | 行为指标驱动路径动态调整 |

### 各智能体详细说明

**内容分析智能体**（`content-analyzer`）

上传资料进入队列处理后，由此智能体负责：
- 调用 `rag` 工具对文档做切分和向量化
- 抽取文档中出现的知识点，与图谱中的 `kp_id` 做映射
- 将映射结果写回 `knowledge_point` 表，丰富知识点的资料来源

**质量评估智能体**（`quality-assessment`）

对资源生成智能体的输出做三段式校验：
1. **检索校验**：用 RAG 检索原始资料，验证生成内容的事实依据
2. **规则校验**：敏感词过滤、格式规范检查
3. **LLM 自检**：调用 `deep_question` 的 validation 阶段；若含代码，用 `code_execution` 实际运行验证

不通过则带原因重试一次，仍失败则标记为 FAILED 并记录原因。

**效果评估智能体**（`effect-assessment`）

收集以下行为指标，存入 `study_behavior` 和 `path_adjustment_log` 表：
- 答题正确率
- 知识点停留时间
- 重复追问次数
- 连续失败次数

触发四条动态调整规则：
- 正确率低 → 插入复习节点
- 编程题连续错 → 补代码案例
- 概念题错多 → 补讲解文档
- 某方向兴趣高 → 补拓展阅读

## 七、开发顺序建议

| 阶段 | 任务 | 验收标准 |
|------|------|----------|
| 1 | 固定课程数据（章节、知识点 ID、依赖关系、种子知识库） | 路径规划能用图谱算顺序 |
| 2 | 登录 + 角色识别 + 角色跳转 | 登录后能识别角色并进入对应页面 |
| 3 | React 基础路由 + 鉴权状态 + WebSocket 封装 | 页面壳子跑通 |
| 4 | Spring Boot 基础目录 + `AiServerClient` 壳子 | 后端能启动并代理 AI 服务 |
| 5 | 跑通 AI 服务四个关键能力（聊天、知识库、题目、答疑） | 能收到流式回答 |
| 6 | 上传队列（表 + worker + 状态回传） | 上传后能看到排队状态 |
| 7 | 最小学习路径（图谱查询 + 路径生成） | 能返回一条学习路径 |

## 八、三个概念不要混淆

| 概念 | 放在哪里 | 负责什么 |
|------|----------|----------|
| 公共知识图谱 | backend | 课程结构导航，路径规划的顺序依据 |
| 公共知识库 | ai-server | 课程讲义、例题、代码，RAG 检索的公共输入 |
| 个人知识库 | ai-server（异步入库） | 用户上传的笔记、作业，个性化答疑的附加上下文 |
