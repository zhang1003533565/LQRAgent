# LQRAgent 项目精简开发指南

## 一、目标是什么

做一个**面向软件杯比赛的 AI 课程学习系统**，课程固定为《Python 高级语言程序设计》。

核心卖点五个：

- 多智能体协同
- 个性化学习路径
- 资源自动生成（含 **生图 / 生视频** 多模态）
- 基于行为反馈的动态调整
- 聊天与资源区统一的多模态展示（文本 + 图片 + 短视频）

**最小闭环（MVP）**：

1. 学生登录，进入聊天式工作台
2. 输入学习目标（如「学习 Python 装饰器」）
3. 系统根据公共知识图谱生成学习路径
4. 点击路径节点，系统生成讲解文档、题目、代码案例，并可生成 **知识点示意图（生图）** 与 **短讲解视频（生视频，可选）**
5. 学生提交答题结果
6. 系统更新画像，调整下一步推荐
7. 学生继续追问，系统结合公共知识库和个人知识库回答

只要这 7 步稳定跑通，软件杯演示就成立。

## 二、架构原则（自研前后端）

### 2.1 三层分工

```
浏览器 (React 5173)  →  Spring Boot BFF (8080)  →  ai-server 能力层 (8001，无 UI)
```

| 层级 | 技术栈 | 核心职责 |
|------|--------|----------|
| 前端 | React + Vite + Zustand | **自研**学生工作台、管理后台；不嵌入 DeepTutor 页面 |
| 后端 | Spring Boot 3 / Java 21 | 鉴权、编排（Supervisor）、画像/路径/队列、**WebSocket 代理**、写库 |
| AI 服务 | Python / FastAPI（DeepTutor 内核） | 大模型、RAG、book/question/solve；**仅 API，不提供比赛演示 UI** |

**调用链**：前端只访问 `/api/*` 与 `/ws/*`（经 Vite 代理到 8080）→ 后端 orchestrator 判意图 → 调对应模块或转发 ai-server → 以统一事件流回传前端 → 更新 MySQL 画像/路径/日志。

> ai-server 能力**不要重写**；协议可参考其 unified_ws，展示层必须在 `frontend/` 自己实现。

### 2.2 成熟模式（对标业界，落地在 Java + React）

| 模式 | 本项目落点 |
|------|------------|
| Supervisor 协调 | `orchestrator` 包：规则/LLM 判意图，再调 specialist |
| 中心化学习者状态 | `learner_profile` 单写；各 Agent 只改授权字段 |
| 编排与 UI 解耦 | 后端推 **事件协议**；前端 Timeline 只渲染事件 |
| 管理端看管道 | `/admin` 看模型、队列、**agent_run_log**；不做聊天 |

### 2.3 边界（明确不做）

- 不使用 `ai-server/web`（DeepTutor 原生 Next 页面）
- 比赛演示不依赖后端控制台 `LQRAgent>`（可 `app.console.enabled=false`）
- 浏览器不直连 `localhost:8001`

## 三、统一事件协议（前后端契约）

WebSocket 端点：`/ws/chat?token=<JWT>`（由后端鉴权并代理 ai-server）。

### 3.1 客户端 → 服务端

```json
{ "type": "message", "content": "我想学 Python 装饰器", "session_id": "可选，UUID" }
```

### 3.2 服务端 → 客户端

| type | 字段 | 用途 | 前端处理 |
|------|------|------|----------|
| `chunk` | `content` | 流式正文 | `appendToLastMessage` |
| `agent_step` | `agent`, `label`, `status`, `detail?` | 智能体步骤 | `AgentTimeline` |
| `artifact` | `kind`, `payload` | 路径/讲义/题目/**媒体**等结构化结果 | 右栏路径 / `ResourceTabs` / 聊天 `MULTI_CARD` |

**`artifact.kind` 约定（含多模态）**

| kind | payload 要点 | 前端 |
|------|----------------|------|
| `learning_path` | 路径节点 JSON | `LearningPathPanel` |
| `lesson` / `quiz` / `code_case` | 文本或结构化题 | `ResourceTabs` |
| `media_image` | `url`, `mime`, `title`, `kpId`, `thumbnailUrl?` | 图片预览 / Tab |
| `media_video` | `url`, `mime`, `title`, `kpId`, `durationSec?`, `posterUrl?` | 视频播放器 / Tab |
| `multi_card` | 卡片数组（文本块 + 媒体块） | `StreamingMessage` 多模态卡片 |
| `profile_patch` | `payload` | 画像增量 | `ProfileSummaryCard` |
| `done` | `session_id?` | 本轮结束 | 结束 streaming |
| `error` | `content` | 错误 | Toast / 消息区 |

`status`：`running` | `done` | `failed`。

### 3.3 智能体标识（`agent` 枚举）

| agent | 中文 | 后端模块 |
|-------|------|----------|
| `orchestrator` | 协调 | `orchestrator` |
| `content_analyzer` | 内容分析 | `contentanalyzer` |
| `path_planner` | 路径规划 | `learningpath` |
| `resource_facade` | 资源生成 | `resourcefacade` |
| `quality_assessment` | 质量评估 | `qualityassessment` |
| `learner_profile` | 学生画像 | `learnerprofile` |
| `qa_agent` | 答疑 | `qaagent` |
| `effect_assessment` | 效果评估 | `effectassessment` |

ai-server 内部 `agent_name` / `stage` 在 `AiServerEventMapper` **一处**映射为上述枚举。

### 4.2 推荐包结构

```
com.lqragent.backend
├── auth / user / security          # 已有
├── admin                           # 已有，扩展 Agent 统计 API
├── systemconfig / aiserver         # 已有 AiServerClient

│   ├── ChatWebSocketHandler        # /ws/chat
│   ├── ChatSessionService
│   └── AiServerWsProxy             # 转发 unified_ws，映射事件
├── learningpath                    # 已有，接图谱
├── learnerprofile                  # 画像 CRUD
├── contentanalyzer                 # 消费 kb_upload_task
├── resourcefacade                  # book / question + 媒体生成编排
├── mediageneration                 # 生图 / 生视频（调第三方或 ai-server 能力）
├── qualityassessment               # 三段校验
├── qaagent                         # deep_solve / chat
├── effectassessment                # study_behavior + 路径调整
├── knowledgegraph                  # 图谱查询（或合入 learningpath）
├── uploadqueue                     # 已有
└── observability
    └── AgentRunLogService          # 写 agent_run_log
```

### 4.3 Orchestrator 第一版（规则路由）

```
用户 message
  ├─ 含「路径/计划/学什么」     → path_planner → artifact(learning_path)
  ├─ 含「出题/练习/测验」       → resource_facade(question)
  ├─ 含「讲义/总结/思维导图」   → resource_facade(lesson) → quality_assessment
  ├─ 含「示意图/配图/思维导图」 → resource_facade(ILLUSTRATION) → mediageneration(image)
  ├─ 含「视频/动画/演示」       → resource_facade(VIDEO_CLIP) → mediageneration(video)
  ├─ 上传任务 COMPLETED 回调    → content_analyzer（异步，写 mapped_kp_ids）
  └─ 默认                      → qa_agent（WS 代理 ai-server）
```

每个分支：`AgentRunLogService.start` → 执行业务 → 推 `agent_step` → 推 `chunk`/`artifact` → `end`。

**管理（ADMIN）**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/status` | 已有 |

| GET | `/api/admin/agent-runs?limit=50` | 最近 agent_run_log |
| GET | `/api/admin/users/{id}/profile-summary` | 某学生画像只读 |

### 4.5 WebSocket 实现要点

1. `ChatWebSocketHandler`：握手校验 JWT（与 `JwtAuthFilter` 一致）。
2. 无 `session_id` 时创建 `chat_session` 记录。
3. `AiServerWsProxy`：连 `sys_config` 中的 `ai-server.ws-url`，把帧转为 3.2 协议。
4. 每条 `agent_step` 同步插入 `agent_run_log`（终态更新 `duration_ms`、`status`）。

## 八、八个智能体与 ai-server 能力

| 智能体 | 后端包 | ai-server 能力 |
|--------|--------|----------------|
| 协调 | `orchestrator` | 不直接调模型；路由 |
| 内容分析 | `contentanalyzer` | `knowledge.py` / RAG |
| 路径规划 | `learningpath` | 图谱本地 + `book` 生成计划文案 |
| 资源生成 | `resourcefacade` | `book` / `question`；媒体类走 `mediageneration` |
| 媒体生成 | `mediageneration` | 生图 API（如 DashScope 万相、OpenAI 等）；生视频 API（如可灵、Runway 等，**可配置**） |
| 质量评估 | `qualityassessment` | validation + `code_execution` |
| 学生画像 | `learnerprofile` | `memory` / `sessions` + MySQL |
| 答疑 | `qaagent` | `unified_ws` / `deep_solve` |
| 效果评估 | `effectassessment` | 行为表 + 规则调路径 |

**内容分析**：完成后写 `kb_upload_task.analysis_result`、`mapped_kp_ids`，并更新图谱关联。

**质量评估**：检索校验 → 规则校验 → LLM/代码执行；失败重试一次。

**效果评估**：正确率低插复习、连续错题补代码案例等，写 `path_adjustment_log`。

### 8.1 多模态媒体生成（生图 / 生视频）

**定位**：在「资源自动生成」下扩展，服务 Python 课程演示——抽象概念配图、关键步骤短视频。

**资源类型（`resource_item.resource_type`）**

| type | 说明 | 典型场景 |
|------|------|----------|
| `ILLUSTRATION` | 静态示意图（生图） | 装饰器调用关系图、数据结构示意 |
| `VIDEO_CLIP` | 短讲解视频（生视频，建议 15～60s） | 知识点动画演示、操作步骤 |

文本类 `LESSON` / `QUIZ` / `CODE_CASE` 不变；媒体类写入 `media_url`、`media_mime`、`generation_prompt`，`content` 可存配套说明 Markdown。

**后端职责划分**

| 组件 | 职责 |
|------|------|
| `MediaGenerationService` | 根据 `kpId` + 课程上下文拼 prompt，调用生图/生视频 API，落盘或对象存储，写 `resource_item` |
| `ResourceFacade` | 统一入口：`generate(kpId, type)`，文本走 ai-server，媒体走 `MediaGenerationService` |
| `QualityAssessment` | 媒体类：敏感词/尺寸/时长校验；失败可重试一次 |
| 管理配置 | `sys_config` 增加 `media.image.*` / `media.video.*`（provider、model、api-key、base-url），与 `llm.*` 并列，管理后台可配 |

**生成流程（路径节点点击）**

```
点击节点 → POST /api/resources/generate { type: ILLUSTRATION | VIDEO_CLIP }
  → path_planner 提供 kp 标题/描述
  → LLM 生成英文/中文绘图 prompt（可选）
  → 调生图/生视频 API → 保存文件 backend/storage/media/{userId}/{uuid}.png|mp4
  → 写 resource_item + 返回 URL
  → 可选 WS 推 artifact(media_image | media_video)
```

**与 ai-server 关系**：优先在 **Java BFF** 直连可配置的生图/生视频 API（演示可控、少改 Python）；若后续需要，可把 `visualize` / `math_animator` 作为备选实现接到 `mediageneration` 适配器。

**前端**：`ResourceTabs` 增加「示意图」「视频」；聊天区收到 `artifact` 或 `MULTI_CARD` 时内联预览，不跳转新页。

**阶段建议**

| 阶段 | 内容 |
|------|------|
| **P1** | `ILLUSTRATION` 生图 + 前端图片 Tab（MVP 演示优先） |
| **P2** | 聊天 `multi_card` 内嵌图片；`quality_assessment` 媒体校验 |
| **P4** | `VIDEO_CLIP` 生视频 + 视频 Tab；管理端媒体 API 配置 |
