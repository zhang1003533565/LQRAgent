## 一、项目目标

做**面向软件杯 A3 赛题的 AI 课程学习系统**，课程固定为《Python 高级语言程序设计》。

### 赛题 5 项核心要求与本项目对应

| # | 赛题要求 | 我们的实现 |
|---|----------|-----------|
| 1 | 对话式学习画像构建（≥6 维度） | `learnerprofile` 智能体 → 前端画像卡片 |
| 2 | 多智能体协同资源生成（≥5 种类型） | 讲义 / 题目 / 代码案例 / 示意图 / 短视频 |
| 3 | 个性化学习路径规划与推送 | `learningpath` + 知识图谱 → 前端路径面板 |
| 4 | 智能辅导-多模态答疑（加分项） | `qaagent` + 聊天多模态卡片 |
| 5 | 学习效果评估-动态调整（加分项） | `effectassessment` + 行为追踪 |

### 最小闭环（MVP）

```
学生登录 → 输入"我想学 Python 装饰器"
→ 系统生成学习路径（路径规划智能体）
→ 点击节点，资源卡片展示讲义/题目/示意图（资源生成智能体）
→ 学生答题，系统更新画像（学生画像智能体）
→ 继续追问，结合知识库回答（答疑智能体）
```

**只要这 6 步跑通，比赛演示就成立。**

## 二、架构总览（新人必读）

### 2.1 三层架构

```
┌──────────────────────────────────────────────────────────────┐
│  浏览器 (React :5173)                                         │
│  · 学生工作台 /workspace    · 管理后台 /admin                   │
└──────────────┬───────────────────────────────────────────────┘
               │  HTTP /api/*    WebSocket /ws/*
               ▼
┌──────────────────────────────────────────────────────────────┐
│  后端 BFF (Spring Boot :8080)                                  │
│  · 鉴权 (JWT)                                                 │
│  · 协调编排 (Orchestrator — 判断意图，调用对应智能体)             │
│  · WebSocket 代理 (转发 ai-server 流式输出给前端)                │
│  · 写 MySQL（画像、路径、答题记录、日志）                         │
└──────────────┬───────────────────────────────────────────────┘
               │  HTTP REST     WebSocket
               ▼
┌──────────────────────────────────────────────────────────────┐
│  AI 能力层 DeepTutor (Python :8001)                            │
│  · 大模型调用（对话、生成）                                      │
│  · RAG 知识库检索                                              │
│  · 题目生成、讲义生成                                           │
│  · 代码执行沙箱                                                │
│  【后端自动拉起，不提供 UI】                                     │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 关键设计原则

1. **前端只访问后端**：浏览器 → `:8080`，永远不直连 `:8001`
2. **后端是"中间人"**：鉴权、编排、代理 ai-server、落库，全在后端做
3. **ai-server 是"发动机"**：只提供 AI 能力 API，不碰业务逻辑和 UI
4. **所有模型配置在管理后台配**：启动后浏览器里填 Key，不需要改配置文件

### 2.3 启动后发生了什么

```
1. 启动后端 (mvn spring-boot:run)
   ├─ 自动执行 init-tables.sql 建表
   ├─ 创建测试账号 admin / student1
   ├─ 自动 pip install ai-server 依赖
   ├─ 自动启动 ai-server (PID 由后端管理)
   └─ 后端 API 可用，但 AI 调用暂不可用（还没配 Key）

2. 启动前端 (npm run dev)
   └─ Vite 开发服务器，/api 和 /ws 代理到 :8080

3. 打开浏览器 http://localhost:5173
   ├─ 用 admin / 123456 登录 → 进入管理后台
   ├─ 在「系统配置」填写大模型 API Key
   ├─ 勾选「同步到 ai-server/.env」→ 保存
   ├─ 点击「测试大模型 API」确认连通
   └─ 重启后端（或等 ai-server 热加载配置）

4. 用 student1 / 123456 登录 → 进入学生工作台
   └─ 开始对话学习
```

## 四、智能体入门（零基础）

### 4.1 什么是多智能体？

把大任务拆成几个"专家"，每个专家只做自己擅长的事，由一个"协调员"统一调度。

**类比**：就像医院里：
- 挂号处（协调员）判断你该去哪个科室
- 内科医生（答疑智能体）看常见病
- 检验科（内容分析智能体）分析化验单
- 药房（资源生成智能体）配药
- 病历系统（画像智能体）记录你的病史

### 4.2 本项目的 9 个智能体

```
用户说："我想学 Python 装饰器"
        │
        ▼
┌──────────────────┐

         ▼

│  learningpath    │     │  (MySQL 里的     │
│                  │     │   知识点依赖关系)  │
└────────┬─────────┘     └─────────────────┘
         │ 生成路径后，用户点击节点
         ▼
┌──────────────────┐
│  资源生成智能体    │  ← 生成讲义/题目/代码/示意图
│  resourcefacade  │
└────────┬─────────┘

    ┌────┴────┐
    ▼         ▼
```

### 4.3 智能体之间怎么通信？

在代码层面，智能体 = 一个 Java Service 类。通信方式：

1. **Orchestrator 调用 Specialist**：直接调用 Service 方法
2. **向前端汇报进度**：通过 WebSocket 推送 `agent_step` 事件
3. **共享数据**：都读写 MySQL 的 `learner_profile` 表

**第一版用"if-else 规则路由"就够**，P4 再考虑 LLM 意图分类。

## 五、三人分工方案

### 5.1 分工表

| 角色 | 负责范围 | 关键文件 |
|------|----------|----------|
| **后端 + 管理端前端** | Spring Boot 所有代码 + 管理后台 UI | `backend/src/main/java/` + `frontend/src/admin/` |
| **前端-学生端** | 聊天、路径、资源、答题、画像、上传 | `frontend/src/student/` |
| **前端-公共层** | 登录页、公共组件、类型、Store、API 封装、路由 | `components/ui/`、`api/`、`store/`、`types/`、`pages/LoginPage.tsx` |

### 5.2 协作约定

1. **API 优先**：后端先定义接口格式 → 公共层写好 API 封装 → 学生端按格式开发 → 后端实现后联调
2. **WebSocket 协议是核心契约**：见第六章，三人必须对齐
3. **Git 分支**：`backend/dev`、`frontend/dev`，功能完成后合入 `main`
4. **类型共享**：前端 `types/agent-events.ts` 必须与后端 WS 事件 JSON 格式一致

## 六、统一事件协议（前后端契约）

> **这是前后端最重要的接口约定。所有 WebSocket 通信必须严格遵循这个格式。**

### 6.1 客户端 → 服务端

```json
{
  "type": "message",
  "content": "我想学 Python 装饰器",
  "session_id": "uuid-可选"
}
```

### 6.2 服务端 → 客户端（7 种事件）

| type | 触发时机 | payload 关键字段 | 前端处理函数 |
|------|----------|-----------------|-------------|
| `chunk` | AI 流式输出每个 token | `content: string` | `appendToLastMessage` |
| `agent_step` | 智能体开始/完成/失败 | `agent, label, status, detail?` | `AgentTimeline` 组件 |
| `artifact` | 智能体产出结构化结果 | `kind, payload` | 根据 kind 分发到路径/资源/媒体面板 |
| `profile_patch` | 画像数据更新 | `payload: partial ProfileSummary` | 头像卡片增量更新 |
| `done` | 本轮对话结束 | `session_id?` | 停止 streaming 动画 |
| `error` | 发生错误 | `content: string` | Toast 提示 + 消息区显示 |

### 6.3 `agent_step` 状态

```json
{ "type": "agent_step", "agent": "path_planner", "label": "正在规划学习路径...", "status": "running" }
{ "type": "agent_step", "agent": "path_planner", "label": "学习路径已生成", "status": "done" }
{ "type": "agent_step", "agent": "path_planner", "label": "路径生成失败", "status": "failed", "detail": "知识图谱未加载" }
```

### 6.4 `artifact.kind` 全量类型

| kind | payload 结构 | 前端目标组件 |
|------|-------------|-------------|
| `learning_path` | `{ goal, nodes[], planDescription }` | `PathSection` |
| `lesson` | `{ kpId, title, content (Markdown) }` | `ResourceSection` text tab |
| `quiz` | `{ kpId, questions[{stem, options[], answer}] }` | `QuizSection` |
| `code_case` | `{ kpId, title, code, explanation }` | `ResourceSection` code tab |
| `media_image` | `{ url, mime, title, kpId, thumbnailUrl? }` | `MediaPreview` / image tab |
| `media_video` | `{ url, mime, title, kpId, durationSec?, posterUrl? }` | `MediaPreview` / video tab |
| `multi_card` | `[{type:"text"|"image"|"video", ...}]` | `MultiCardMessage` 聊天内嵌卡片 |
| `profile_patch` | `Partial<ProfileSummary>` | `ProfileCard` 增量更新 |

### 6.5 智能体标识枚举

```java
public enum AgentId {
    orchestrator,      // 协调
    content_analyzer,  // 内容分析
    path_planner,      // 路径规划
    resource_facade,   // 资源生成
    quality_assessment,// 质量评估
    learner_profile,   // 学生画像
    qa_agent,          // 答疑
    effect_assessment  // 效果评估
}
```

前端对应 `types/agent-events.ts` 里的 `AgentId` 类型，两边必须一致。

## 七、ai-server 接口说明（我们需要的部分）

> DeepTutor 开源项目提供的接口很多，我们只用到以下这些。

### 7.1 我们使用的接口

| 用途 | 方法 | 路径 | 后端调用方 |
|------|------|------|-----------|
| 健康检查 | GET | `/api/v1/knowledge/health` | `AiServerClient.ping()` ✅ |
| 创建知识库 | POST | `/api/v1/knowledge/kb` | `AiServerClient.createKnowledgeBase()` ✅ |
| 上传文档 | POST | `/api/v1/knowledge/{kb_id}/documents` | `AiServerClient.uploadDocument()` ❌ P3 |
| 生成题目 | POST | `/api/v1/question/generate` | `AiServerClient.generateQuestion()` 🟡 |
| 生成讲义 | POST | `/api/v1/book/generate` | `AiServerClient.generateLesson()` 🟡 |
| 流式对话 | WS | `/api/v1/ws` | `AiServerWsProxy` ✅ |

### 7.2 我们不用的接口

- ai-server 的 Next.js Web UI（`ai-server/web/`）— 我们自研前端
- CLI 模式（`deeptutor chat`）— 我们只走 API/WS
- Matrix/即时通讯集成（TutorBot）— 比赛不涉及
- GeoGebra 数学可视化 — 课程是 Python，不需要
- 论文搜索（arXiv）— 不需要
- 协作写作、图书编辑器 — 不需要

### 7.3 ai-server .env 配置（后端自动管理）

```
LLM_BINDING=openai
LLM_MODEL=gpt-4o-mini
LLM_API_KEY=sk-xxx
LLM_HOST=https://api.openai.com/v1
EMBEDDING_BINDING=openai
EMBEDDING_MODEL=text-embedding-3-large
EMBEDDING_API_KEY=sk-xxx
EMBEDDING_HOST=https://api.openai.com/v1/embeddings
```
