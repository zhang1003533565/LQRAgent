## 一、架构概览

### 1.1 技术栈

React 19 + Vite 5 + TypeScript + Zustand(状态管理) + axios

## 二、API 封装层

| 模块 | 文件 | 对接后端 | 状态 |
|------|------|---------|------|
| 认证 | `shared/api/auth.ts` | POST /api/auth/login, /register | ✅ |
| 学习路径 | `api/student/learningPath.ts` | GET /api/learning-path, /current | ✅ |
| 资源 | `api/student/resources.ts` | GET /api/resources/{kpId}, POST /generate | ✅ (catch有占位数据) |
| 答题 | `api/student/quiz.ts` | GET /api/resources/{kpId}(QUIZ), POST /quiz/submit | ✅ |
| 画像 | `api/student/profile.ts` | GET /api/profile/summary | 部分 (catch有占位数据) |
| 上传 | `api/student/upload.ts` | POST /api/upload, GET /upload/tasks | ✅ |
| HTTP | `shared/api/http.ts` | axios实例, baseURL=/api, 自动JWT, 401跳转 | ✅ |

**注意：** resources 和 profile 的 catch 分支返回硬编码占位数据，联调时需替换为错误提示。

## 三、WebSocket 事件处理

`useWebSocket` hook 连接 `ws://host/ws/chat`，接收后端6种事件：

| type | 前端处理 | 关联 Store |
|------|---------|-----------|
| `chunk` | `chatStore.appendToLastMessage` | chatStore |
| `agent_step` | `agentTraceStore.upsert` | agentTraceStore |
| `artifact` | 按kind分发到对应面板 | artifactStore / pathStore |
| `profile_patch` | `profileStore.patch` | profileStore |
| `done` | `chatStore.setStreaming(false)` | chatStore |
| `error` | Toast提示 | — |

**artifact.kind → 目标组件映射：**

| kind | 目标 |
|------|------|
| `learning_path` | PathSection 路径面板 |
| `lesson` | ResourceSection 讲义Tab(Markdown) |
| `quiz` | QuizSection 答题面板 |
| `code_case` | ResourceSection 代码Tab(语法高亮) |
| `mind_map` | ResourceSection 思维导图 |
| `extended_reading` | ResourceSection 拓展阅读Tab |
| `media_image` | MediaPreview 图片 |
| `media_video` | MediaPreview 视频 |
| `multi_card` | MultiCardMessage 图文卡片 |
| `profile_patch` | ProfileCard 画像卡片 |

## 五、开发规划

### Phase 1：聊天完善 + Artifact卡片（1-2天）

| 任务 | 内容 |
|------|------|
| 1.1 artifact卡片 | 监听artifact事件，按kind在聊天流中插入对应卡片组件 |
| 1.2 步骤条联动 | 确认agentTraceStore.upsert + AgentTimeline正确显示多Agent执行进度 |
| 1.3 Mermaid渲染 | 引入mermaid-js库，渲染答疑Agent返回的流程图代码 |

### Phase 2：学习路径页面（1-2天）

| 任务 | 内容 |
|------|------|
| 2.1 路径可视化 | pathStore.nodes → 节点流程图(CSS Grid/Flexbox + SVG连线，或react-flow) |
| 2.2 节点点击交互 | 点击节点 → selectedKpId → 触发资源生成 → 显示资源 |
| 2.3 状态更新 | 完成学习后节点状态COMPLETED → 下一个ACTIVE |
| 2.4 刷新恢复 | 页面加载时调GET /api/learning-path/current恢复路径 |

### Phase 3：资源展示页面（1-2天）

| 任务 | 内容 |
|------|------|
| 3.1 Tab切换 | 讲义(Markdown)/代码(prism.js高亮)/题目/思维导图/拓展阅读 |
| 3.2 资源加载 | 点击节点 → GET /api/resources/{kpId}?type=XXX → 渲染 |
| 3.3 生成触发 | 无资源时显示"生成"按钮 → POST /api/resources/generate |

### Phase 4：答题页面（1天）

| 任务 | 内容 |
|------|------|
| 4.1 题目展示 | GET /api/resources/{kpId}?type=QUIZ → 选择题/填空题UI |
| 4.2 答题交互 | 选择答案 → POST /api/quiz/submit → 显示得分+解析 |
| 4.3 联动画像 | 提交后显示"已更新学习画像"提示 |

### Phase 5：画像 + 上传 + 细节（1-2天）

| 任务 | 内容 |
|------|------|
| 5.1 画像卡片 | 调GET /api/profile/summary填充6维度数据 + WS patch实时刷新 |
| 5.2 上传结果 | 上传完成后展示分析摘要+映射知识点 |
| 5.3 Mock清理 | 移除API catch中的硬编码占位数据，改为错误提示+空状态 |
| 5.4 UI细节 | 空状态引导、Skeleton加载态、错误Toast、响应式适配 |

### Phase 6：联调验证（2-3天，与后端协作）

> MVP闭环6步：登录 → 输目标 → 出路径 → 生资源 → 做题 → 答疑

| 任务 | 内容 |
|------|------|
| 6.1 WS全链路 | 确认6种事件前后端对齐，artifact正确渲染 |
| 6.2 闭环验证 | 按演示脚本走通6步，不需手动干预 |
| 6.3 多会话 | 侧栏会话列表+切换(如时间允许) |
