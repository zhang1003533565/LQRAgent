# LQRAgent 启动指南

本文说明如何在本地从零启动 **LQRAgent**（个性化学习系统）。适用于 Windows / macOS / Linux。

---

## 一、项目结构

```
LQRAgent/
├── backend/          # Spring Boot 业务后端（端口 8080）
├── frontend/         # React + Vite 前端（端口 5173）
├── ai-server/        # Python DeepTutor AI 服务（端口 8001）
└── docs/
    ├── schema.sql    # 完整数据库脚本（可选手动执行）
    └── 启动指南.md   # 本文
```

**调用关系：**

```
浏览器 → frontend:5173 → 代理 /api、/ws → backend:8080 → ai-server:8001（大模型/RAG）
```

---

## 二、环境要求

| 工具 | 版本建议 | 用途 |
|------|----------|------|
| **JDK** | 21（推荐）或 22～25 | 编译运行后端 |
| **Maven** | 3.9+ | 构建后端 |
| **Node.js** | 18+ / 20 LTS | 运行前端 |
| **MySQL** | 8.0+ | 业务数据库 |
| **Python** | 3.10+ | AI Server（后端可自动拉起） |

> **说明**：项目编译目标为 Java 21。若使用 JDK 25，需 Lombok 1.18.42+（已在 `pom.xml` 中配置）。

## 三、数据库准备

### 3.1 创建库

在 MySQL 中执行（或使用客户端工具）：

```sql
CREATE DATABASE IF NOT EXISTS agent
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 3.2 修改连接信息

编辑 `backend/src/main/resources/application.properties`：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/agent?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=123456   # 改成你的 MySQL 密码
```

### 3.3 表结构如何生成

**方式 A（推荐）**：首次启动后端时自动执行

- `spring.sql.init.mode=always` 会运行 `backend/src/main/resources/db/init-tables.sql`
- JPA 会为 `sys_user`、`kb_upload_task` 等实体同步表结构

**方式 B**：手动执行完整脚本

```bash
mysql -u root -p < docs/schema.sql
```

启动后在数据库中应能看到约 **12 张表**（如 `sys_user`、`sys_config`、`chat_message`、`learning_path` 等）。

## 四、配置大模型 API（必做）

对话、RAG、资源生成依赖 **ai-server** 中的大模型配置。

### 方式 1：管理后台（推荐）

1. 先完成第五节启动后端与前端  
2. 使用管理员登录 → **系统配置**  
3. 在 **「大模型对话 API」** 中填写：
   - 提供商（如 `openai`、`deepseek`、`dashscope`）
   - 模型名称（如 `gpt-4o-mini`、`deepseek-chat`）
   - API Key
   - API 地址（如 `https://api.openai.com/v1`）
4. 勾选 **同步到 ai-server/.env**，点击 **保存模型配置**  
5. 点击 **测试大模型 API** 验证  
6. **重启后端**（或仅重启 AI 进程）使 `.env` 完全生效  

### 方式 2：直接编辑 ai-server/.env

```bash
cd ai-server
copy .env.example .env    # Windows
# cp .env.example .env    # macOS / Linux
```

编辑 `.env` 中至少配置：

```env
LLM_BINDING=openai
LLM_MODEL=gpt-4o-mini
LLM_API_KEY=你的密钥
LLM_HOST=https://api.openai.com/v1

EMBEDDING_BINDING=openai
EMBEDDING_MODEL=text-embedding-3-large
EMBEDDING_API_KEY=你的密钥
EMBEDDING_HOST=https://api.openai.com/v1/embeddings
```

## 五、启动步骤

建议开 **3 个终端**（若 `ai-server.auto-start=true`，终端 3 可省略）。

### 5.1 启动后端

```powershell
cd LQRAgent\backend
mvn spring-boot:run
```

成功标志：

- 日志出现 `Started BackendApplication`
- 监听 **http://localhost:8080**
- 自动创建测试账号（见下文）
- 若 `ai-server.auto-start=true`，会安装依赖并启动 Python 服务（日志见 `backend/logs/ai-server.log`）

**启动后控制台（可选）**

同一终端出现 `LQRAgent>` 提示符时，可输入 `help` 查看管理命令（配置、用户、上传队列等）。关闭菜单输入 `quit`，**不会**停止 Web 服务。

禁用控制台：在 `application.properties` 中设置 `app.console.enabled=false`。

### 5.2 启动前端

```powershell
cd LQRAgent\frontend
npm install
npm run dev
```

浏览器访问：**http://localhost:5173**

开发模式下，`/api` 与 `/ws` 会代理到 `http://localhost:8080`（见 `frontend/vite.config.ts`）。

### 5.3 单独启动 AI Server（可选）

若已关闭自动启动（`ai-server.auto-start=false`），可手动：

```powershell
cd LQRAgent\ai-server
python -m pip install -e ".[server]"
python -m deeptutor.api.run_server
```

默认地址：**http://localhost:8001**

## 八、配置说明

### 8.1 配置文件位置

| 配置类型 | 位置 | 说明 |
|----------|------|------|
| 后端默认配置 | `backend/src/main/resources/application.properties` | 数据库、端口、JWT 等 |
| 运行时覆盖 | MySQL 表 `sys_config` | 管理后台保存后优先 |
| 大模型 API | 管理后台 → 系统配置 | 同步到 `ai-server/.env` |
| AI 服务入口 | `ai-server.base-url` | 默认 `http://localhost:8001` |

### 8.2 常用配置键

| 键 | 含义 |
|----|------|
| `ai-server.base-url` | LQRAgent 连接 Python 服务的 HTTP 地址 |
| `ai-server.ws-url` | AI WebSocket 地址（预留） |
| `ai-server.auto-start` | 是否随后端自动启动 AI Server |
| `llm.binding` / `llm.model` / `llm.api-key` / `llm.host` | 大模型 API |
| `embedding.*` | 嵌入模型 API（知识库） |

## 九、常见问题

### 9.1 端口 8080 被占用

```powershell
netstat -ano | findstr ":8080"
taskkill /PID <进程ID> /F
```

或修改 `application.properties` 中 `server.port`，并同步修改 `frontend/vite.config.ts` 代理目标。

### 9.2 编译报错 `TypeTag :: UNKNOWN`

JDK 过新而 Lombok 过旧。请确认 `backend/pom.xml` 中 `lombok.version` 为 **1.18.42**，然后：

```powershell
cd backend
mvn clean compile
```

### 9.3 登录提示用户名或密码错误

- 确认使用 `admin` / `123456` 或 `student1` / `123456`  
- 重启后端以触发账号同步  
- 检查 MySQL 中 `sys_user` 表是否有数据  

### 9.4 数据库只有 2 张表

需执行完整初始化：重启后端（会跑 `init-tables.sql`），或手动执行 `docs/schema.sql`，然后在客户端 **刷新** 表列表。

### 9.5 工作台显示「连接中…」

当前 **WebSocket 聊天代理尚未实现**（`/ws/chat`），属于已知待开发项，不影响登录与管理后台。

### 9.6 终端中文乱码（Windows）

项目已配置 UTF-8。若仍乱码，在终端执行 `chcp 65001` 后重新启动后端。

### 9.7 AI Server 启动失败

- 查看 `backend/logs/ai-server.log`  
- 确认 Python 3.10+ 已安装且在 PATH 中  
- 在 `ai-server` 目录手动执行 `python -m pip install -e ".[server]"`  
- 检查 `.env` 中 LLM 配置是否完整

## 十一、生产部署提示（简述）

1. 前端：`npm run build`，将 `dist/` 用 Nginx 托管，并反代 `/api`、`/ws` 到后端。  
2. 后端：`mvn package` 生成 jar，配置环境变量或外部 `application-prod.properties`。  
3. 修改默认 JWT 密钥、MySQL 密码、关闭 `app.console.enabled`。  
4. AI Server 建议独立进程 + 进程守护，不要依赖开发模式自动拉起。
