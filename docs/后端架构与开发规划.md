## 一、系统架构

### 1.1 三层架构

```
浏览器 (React :5173)
  · 学生工作台 /workspace    · 管理后台 /admin
       │ HTTP /api/*    WebSocket /ws/*
       ▼
Spring Boot 后端 (:8080)  ← 本文档范围
  · JWT 鉴权                · 智能体编排 (Orchestrator)
  · WebSocket 代理          · MySQL 读写 (14张表)
       │ HTTP / WS
       ▼
Python AI Server (:8001)
  · LLM 调用 (DeepSeek)    · RAG 知识库检索
  · 题目/讲义生成           · 代码执行沙箱
```

技术栈：Spring Boot 3.2.5 + MySQL 8 + Redis（预留）+ JDK 21 + Maven

### 1.2 核心设计原则

- 前端只访问后端 `:8080`，不直连 `:8001`
- 后端是"中间人"：鉴权、编排、代理 ai-server、落库
- ai-server 是"发动机"：只提供 AI 能力 API，不碰业务逻辑
- 所有模型配置在管理后台配（运行时可改，不用改配置文件）
- LLM 不可用时自动降级到规则/模板兜底

### 1.3 启动流程

```
mvn spring-boot:run
  ├─ 执行 init-tables.sql 建14张表
  ├─ 创建测试账号 admin/123456、student1/123456
  ├─ 自动 pip install + 拉起 ai-server
  └─ 后端就绪，等待前端连接

npm run dev (前端)
  └─ Vite :5173, /api 和 /ws 代理到 :8080
```

## 二、Agent 引擎架构

### 2.1 LLM + Tool Calling 推理循环

```
AgentEngine.run(agent, task):
  messages = [systemPrompt, userInput]
  while (round < 10):
    response = LLM.chat(messages, tools=agent.getTools())
    if response 是纯文本 → return 最终答案
    if response 含 tool_calls:
      for call in tool_calls:
        result = ToolRegistry.execute(call)
        messages += result (截断到500字符)
    → 继续下一轮
```

LLM 未配置时自动降级到 `agent.process(task)`（switch/case 旧模式）。

### 2.2 8 个智能体 & 20 个工具

| Agent | 角色 | Tools | 关键 Service | 状态 |
|-------|------|-------|-------------|------|
| **Orchestrator** | 意图分类+路由 | route_intent, greet, handle_help, handle_learning_path, handle_resource_generate, handle_media_generate | OrchestratorService (LLM+关键词双保险分类) | 工具已注册，process()保留降级 |
| **LearnerProfile** | 6维画像提取 | get_profile, extract_profile | ProfileExtractor (LLM抽取+DB+WS推送) | 完整 |
| **LearningPath** | BFS路径规划 | generate_path | LearningPathService (31点60边BFS) | BFS完成，LLM排序待验证 |
| **ResourceFacade** | 5种资源生成 | generate_lesson/quiz/code_case/mind_map/extended_reading | ResourceLlmGenerator (LLM+模板兜底) | 完整，含质检重试 |
| **QualityAssessment** | 4层质检 | assess_full | FormatChecker + LLMVerifier + SensitiveFilter + AcademicChecker | 完整 |
| **EffectAssessment** | 薄弱点+路径调整 | evaluate_quiz, analyze_weakness | EffectAssessmentService (行为追踪+LLM分析+复习节点插入) | 完整 |
| **IntelligentQA** | 流式答疑 | answer, generate_mermaid | QaAgentService (WS流式+Mermaid生成) | WS通道需适配 |
| **ContentAnalyzer** | 文档分析 | analyze | ContentAnalyzerService (关键词+图谱匹配) | 完整 |
| **MediaGeneration** | 生图/视频 | — (独立Controller) | MediaGenerationService (DALL-E 3/SD3/mock) | 图片OK，视频预留 |

### 2.3 关键数据流

**资源生成：**
```
POST /api/resources/generate → ResourceFacadeService.generate()
  → LLM 生成 → QualityAssessment.assessFull()
  → 合格 → 存 resource_item 表
  → 不合格 → 删除重试(最多1次) → 仍失败返回模板兜底
```

**答题+效果评估：**
```
POST /api/quiz/submit → QuizService.submit()
  → 评分 → 更新 learner_profile
  → EffectAssessment.evaluateQuizResult()
    → 低分 → learning_path_step 表插入复习节点
  → 记录 study_behavior
```

**学习路径：**
```
GET /api/learning-path?goal=xxx → LearningPathService.generatePath()
  → 解析 goal → BFS 在 knowledge_point/knowledge_edge 有向图上找最短路径
  → (待接入) LLM 根据 learner_profile 排序
  → 存 learning_path + learning_path_step
```

## 三、数据库 (14张表)

| 表 | 用途 | 备注 |
|----|------|------|
| `sys_user` | 用户(含角色) | 预置 admin + student1 |
| `sys_config` | 运行时配置 | 可覆盖 application.properties |
| `knowledge_point` | 知识图谱节点 | 31条(Python入门→高级) |
| `knowledge_edge` | 图谱前置关系 | ~60条有向边 |
| `learner_profile` | 6维学生画像 | knowledge_level/learning_goal/cognitive_style/common_errors/learning_pace/interest_direction |
| `learning_path` | 学习路径 | user_id + goal |
| `learning_path_step` | 路径步骤 | kp_id + order + status(PENDING/ACTIVE/COMPLETED/SKIPPED) |
| `resource_item` | 教学资源 | LESSON/QUIZ/CODE_CASE/MIND_MAP/EXTENDED_READING + quality_score |
| `quiz_record` | 答题记录 | score + is_correct |
| `study_behavior` | 行为追踪 | action(VIEW/CHAT/QUIZ/UPLOAD) + duration_sec |
| `path_adjustment_log` | 路径调整记录 | trigger(LOW_SCORE/REPEAT_ERROR) + action(INSERT_REVIEW) |
| `chat_session` | 聊天会话 | user_id + title |
| `chat_message` | 对话消息 | session_id + sender(USER/AI) + body |
| `kb_upload_task` | 文档上传队列 | status(PENDING/PROCESSING/COMPLETED/FAILED) + analysis_result |
| `agent_run_log` | Agent运行日志 | agent + intent + status + duration_ms |

## 四、WebSocket 事件协议（前后端契约）

### 客户端 → 服务端

```json
{ "type": "message", "content": "...", "session_id": "uuid" }
```

### 服务端 → 客户端（6种事件）

| type | 触发时机 | payload | 前端处理 |
|------|---------|---------|---------|
| `chunk` | LLM流式每个token | `content: string` | 追加到最后一条消息 |
| `agent_step` | Agent开始/完成/失败 | `agent, label, status, detail?` | 步骤条组件 |
| `artifact` | 结构化结果 | `kind, payload` | 按kind分发卡片 |
| `profile_patch` | 画像更新 | `payload: Partial<ProfileSummary>` | 卡片增量刷新 |
| `done` | 本轮结束 | `session_id?` | 停止loading |
| `error` | 出错 | `content: string` | Toast提示 |

**artifact.kind 类型：** `learning_path` / `lesson` / `quiz` / `code_case` / `mind_map` / `extended_reading` / `media_image` / `media_video` / `multi_card` / `profile_patch`

## 七、开发规划

### Phase 1：Agent 链路调试（3-5天）

> 目标：让8个Agent在LLM推理循环中真正跑通

| 任务 | 内容 | 验证方法 | 预估 |
|------|------|---------|------|
| 1.1 Orchestrator路由 | 验证LLM意图分类正确性 | 发5种意图消息，检查日志中tool_calls | 1天 |
| 1.2 子Agent派发 | Orchestrator → 子Agent完整调用链 | 日志出现完整链：Orchestrator → LearningPathAgent → generate_path | 1天 |
| 1.3 资源生成链路 | 5种资源LLM生成+质检+降级 | resource_item表有记录，quality_score有值 | 1天 |
| 1.4 答疑WS链路 | QaAgent流式回答能正常返回 | 提问后收到流式文字 | 0.5天 |
| 1.5 画像提取 | 对话中自动提取6维画像 | learner_profile表有记录 | 0.5天 |
| 1.6 效果评估联动 | 答题→薄弱点→路径插入复习节点 | path_adjustment_log有记录，路径多了复习步骤 | 0.5天 |

### Phase 2：向量化/RAG 完善（3-4天）

> 目标：让知识库的向量化和 RAG 检索真正能用。详见 [向量化RAG完善方案](./向量化RAG完善方案.md)

| 任务 | 内容 | 预估 |
|------|------|------|
| 2.1 修复 Embedding 配置 | 测试连通性 + reindex kb-public + AiServerClient 加方法 | 0.5天 |
| 2.2 扩展 AiServerClient | 加 searchKB/files/progress/reindex/status 5 个方法 + 修复超时 | 0.5天 |
| 2.3 透传 RAG 引用来源 | StreamCallback 加 onSources + WS 推 sources artifact 给前端 | 1天 |
| 2.4 改造 ContentAnalyzer | PDFBox 解析 PDF + RAG 检索替代子串匹配 | 1天 |
| 2.5 上传进度追踪 | processTask 加 getProgress 轮询，等向量化完成再标 COMPLETED | 0.5天 |

### Phase 3：增强与补全（2-3天）

> 目标：填补已知缺陷，增强鲁棒性

| 任务 | 内容 |
|------|------|
| 3.1 LLM个性化排序 | LearningPathService BFS后接LLM排序（三级降级：LLM→画像规则→纯BFS） |
| 3.2 QaAgent适配 | answer工具的WS通道与AgentEngine推理循环兼容 |
| 3.3 质检规则库填充 | SensitiveFilter敏感词列表 + AcademicChecker学术规范规则 |
| 3.4 媒体生成配置 | DALL-E 3 / SD3 API Key配置+测试，视频接口预留 |

### Phase 4：前后端联调（3-4天，与前端协作）

> 目标：MVP闭环6步跑通

```
登录 → 输入学习目标 → 生成路径 → 点击节点生成资源 → 做题 → 追问答疑
```

| 任务 | 内容 |
|------|------|
| 4.1 WS事件联调 | 确认6种事件(chunk/agent_step/artifact/profile_patch/done/error)前后端对齐 |
| 4.2 artifact卡片推送 | 后端生成路径/资源时正确推送artifact事件 |
| 4.3 端到端验证 | 按演示脚本走通6步，中间不需手动干预 |

### Phase 5：打磨与演示准备（2-3天）

| 任务 | 内容 |
|------|------|
| 5.1 管理后台Agent监控 | agent_run_log统计面板(调用次数/成功率/耗时) |
| 5.2 演示数据脚本 | SQL脚本重置student1数据(保留知识图谱和配置) |
