# LQRAgent 项目文档总结

> 本文档汇总了项目开发过程中的设计文档和规划文档，供开发参考。

---

## 一、前端架构

### 技术栈
- React 19 + Vite 5 + TypeScript + Zustand + axios

### 目录结构
```
frontend/src/
├── app/                # 路由壳
├── components/         # 组件
│   ├── admin/          # 管理后台组件
│   └── student/        # 学生工作台组件
├── pages/              # 页面
├── utils/              # 工具函数、hooks、stores
└── api/                # API 接口
```

### 主要功能
- 聊天学习（WebSocket 流式对话）
- 学习路径（可视化节点图）
- 学习资源（讲义、练习题）
- 学习画像（个人中心）
- 知识图谱（管理后台）

---

## 二、后端架构

### 技术栈
- Java 21 + Spring Boot 3.2 + MySQL + Redis + JPA

### 三层架构
```
浏览器 (React :5173)
     ↓ HTTP /api/*    WebSocket /ws/*
Spring Boot 后端 (:8080)
     ↓
MySQL (业务数据) + Redis (会话/缓存) + ai-server (AI推理)
```

### 核心模块
- **OrchestratorCore** — 调度中枢，意图识别 + 任务分发
- **BaseAgent** — Agent 基类，LLM 推理循环
- **7 个智能体** — Profile, LearningPath, Resource, Quality, Effect, QA, Content
- **LlmClient** — LLM API 客户端
- **Redis Streams** — Agent 间消息队列

---

## 三、多智能体架构

### 设计原则
- FIPA ACL + Google A2A 规范
- Orchestrator 作为全局调度中枢
- 通过 Redis Streams 异步通信

### Agent 列表
| Agent | 功能 | 状态 |
|-------|------|------|
| ProfileAgent | 学习画像分析 | ✅ |
| LearningPathAgent | 学习路径规划 | ✅ |
| ResourceAgent | 学习资源生成 | ✅ |
| QualityAgent | 资源质量评估 | ✅ |
| EffectAgent | 学习效果分析 | ✅ |
| QaAgent | 智能问答 | ✅ |
| ContentAgent | 内容分析 | ✅ |

---

## 四、RAG 向量化方案

### 核心思路
- Java 后端不自己实现向量化
- 全部调 ai-server 现成接口

### 实施要点
- 使用 SiliconFlow Embedding API
- 知识库存储在 ai-server
- 检索结果注入 LLM 上下文

---

## 五、实施约束

### 铁律
1. 改之前必须读懂现有代码
2. 一次只改一个东西
3. 不删现有功能，只迁移、重构、新增
4. 每个改动必须有验证方法
5. 遇到不确定的地方，停下来问

---

## 六、关键设计决策

### 1. ChatView 聊天
- 使用 OrchestratorCore 做意图识别
- QA 走流式对话（QaAgentService）
- 学习路径走同步生成（LearningPathService）

### 2. 学习路径生成
- 优先匹配已有知识图谱（31 个 Python 知识点）
- 无匹配时动态生成（LLM 直接生成路径）

### 3. 前端布局
- 响应式网格布局
- 流式输出时使用 instant 滚动避免抖动
